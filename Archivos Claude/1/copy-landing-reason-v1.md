# COPY LANDING — REASON v1

> Texto completo de la landing de Reason. 7 secciones, todas escritas con audiencia primaria "kinesiólogo joven 1-3 años", ángulo B+C ("decisiones, no contenido" con voz del autor presente) y tono híbrido (declarativo en hero/CTAs, reflexivo en explicaciones).
>
> **Cómo usar este documento:** sirve como referencia editorial permanente y como input para Antigravity cuando construyas la landing. Cada sección tiene titular, subtitular, cuerpo (cuando aplica), CTA y notas de diseño.

## SECCIÓN 3 — CÓMO FUNCIONA

**Titular**

Cada contenido te lleva a una decisión.

**Subtitular**

Estructura fija. Lectura de 2 a 3 minutos. Visualizaciones cuando ayudan a pensar mejor que el texto.

**Bloque 1 — Estructura fija**

Cada contenido sigue la misma arquitectura: qué tenés que saber, interpretación clínica, aplicación práctica, qué evitar, conclusión accionable. Sabés dónde encontrar cada cosa, siempre.

**Bloque 2 — Formato breve**

Diseñado para leer entre paciente y paciente. Si te lleva más de tres minutos, está mal escrito.

**Bloque 3 — Visualizaciones**

Árboles de decisión, tablas comparativas, líneas de tiempo. Cuando un razonamiento se ve, se aplica mejor que cuando se lee.

**Notas de diseño:**
Tres bloques horizontales en grid de tres columnas. Cada bloque con un mini-elemento visual arriba (esquema de los 6 bloques editoriales en miniatura, número grande "2-3 min" en Geist Mono, thumbnail de un árbol). Los bloques son cards con borde 0.5px, radio 12px, padding 24px. Sin íconos genéricos; los elementos visuales son específicos del producto.

## NOTAS GENERALES PARA TODA LA LANDING

**Tipografía aplicada:**
- Hero: Geist 56-64px peso 500 para titular, 18-20px peso 400 para subtitular
- Titulares de sección: Geist 32-40px peso 500
- Subtitulares: Geist 18px peso 400, color `--color-text-secondary`
- Cuerpo: Geist 16px peso 400, color `--color-text-primary`
- Microcopy y etiquetas: Geist 12px peso 500 mayúsculas, color `--color-text-secondary`
- Datos numéricos (precios, tiempos): Geist Mono

**Color aplicado:**
- Acento terracota en CTAs primarios y elementos destacados (card anual, divisores en preview)
- Fondos alternados sutilmente entre `--color-bg-primary` y `--color-bg-secondary` para separar secciones
- Sección 7 con `--color-bg-tertiary` para cierre visual claro

**Espaciado aplicado:**
- Padding vertical entre secciones: 96-120px en desktop, 64-80px en mobile
- Padding interno de cada sección: 32-48px
- Max-width del contenedor de texto: 720px (mantiene legibilidad)

**Consistencia con el sistema visual v1.1:**
Toda la landing respeta el sistema visual definido para Reason. No hay decisiones de diseño que rompan el sistema. Si algo necesita romper la regla, primero se actualiza el sistema visual.
